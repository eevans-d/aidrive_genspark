# Security Policy

## Reporting a Vulnerability

Please report security issues **privately**.

Preferred channels:
1) GitHub Security Advisory (recommended for private disclosure).
2) Repository owner via GitHub (see `CODEOWNERS`).

## What to Include
- Clear description of the issue and impact.
- Steps to reproduce (minimal and safe).
- Affected files/paths or components.
- Suggested fix (if available).

## Do Not
- Do not post vulnerabilities in public issues.
- Do not include secrets or real credentials in reports.

## Service Level
This project does not currently guarantee response times. Reports will be triaged as soon as possible.
